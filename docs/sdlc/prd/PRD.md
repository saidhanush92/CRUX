# PRD: Crux v1.0

**Status:** draft (awaiting spec-critic + HITL approval)
**Owner:** Dhanush Sai
**Derived from:** IDEA-001 (the Crux product note)
**Last updated:** 2026-05-05

## Goals

- Deliver a runtime-neutral SDLC orchestrator that walks every software idea through 8 disciplined gates, with AI doing the work and humans approving the decisions that matter.
- Ship gates 1 through 7 end-to-end at v1.0 (input, PRD, modules, architecture, harness, design, build), so a real user project can complete a full feature pass without missing-feature errors. (REQ-CRUX-001)
- Prove the runtime-neutrality thesis is structurally sound by shaping a small adapter interface (15–20 functions, 7 concern groups) and implementing it once for Claude Code at v1.0. (REQ-CRUX-005, REQ-CRUX-006, REQ-CRUX-007)
- Make the audit trail load-bearing and portable: trace graph stored as plain markdown / YAML in the user's repo, queryable via slash commands and a render-graph script. (REQ-CRUX-003, REQ-CRUX-004, REQ-CRUX-019)
- Make the system smarter on every incident: the INC → CHG → AMD cascade lands amendments as layered files that subsequent agent invocations honor automatically. (REQ-CRUX-010, REQ-CRUX-018)
- Keep the founding-team workflow (PM + 1–2 developers) honest and fast: compressed-mode default, advisory harness install, per-task cost halt, no surface bloat. (REQ-CRUX-002, REQ-CRUX-008, REQ-CRUX-011, REQ-CRUX-016)

## Non-goals

The following are explicit non-goals at v1.0 and must not block the release:

- **Gate 8 deploy automation.** No `/crux-deploy`, no GitHub Actions deploy workflows, no release-cut tooling. The user ships using whatever mechanism their team already uses. (deferred to v1.1) — derived from GRILL-CRUX-001
- **Loop 9 listener.** No background watcher, no production telemetry integration, no CI-failure auto-detector. Incidents are opened manually via `/crux-incident report`. (deferred to v1.1; CI-failure detection is the most likely first source) — derived from GRILL-CRUX-007
- **Hosted audit-site.** `packages/audit-site` ships as a stub README only. No Astro app, no GitHub Pages deployment, no `audit-site-deploy.yml` CI workflow. (deferred to v1.1) — derived from GRILL-CRUX-015
- **Automated PR creation.** `/crux-task` produces a `PR_DESCRIPTION.md` but does not invoke `gh pr create` or any equivalent. The user opens the PR manually. (deferred to v1.1; likely as `/crux-pr`) — derived from GRILL-CRUX-019
- **REL artifact format.** `docs/sdlc/releases/` exists as a placeholder; no `REL-*.yaml` files are produced at v1.0. (deferred to v1.1 alongside actual deploy automation) — derived from GRILL-CRUX-020
- **Deterministic harness install.** Gate 5 install is advisory at v1.0; cross-team byte-identical reproduction of the harness is a v1.1+ goal. — derived from GRILL-CRUX-012
- **Skill-declared `conflicts_with` field.** Hook-collision resolution is hard-halt-only at v1.0; declarative skill-level conflict declarations are v1.1+. — derived from GRILL-CRUX-018
- **Cross-model subagent assignment.** Coder-on-Sonnet / reviewer-on-Opus role-based model selection is a v1.1+ optimization. — derived from GRILL-CRUX-013

## Personas

- **Primary — small founding team (PM + 1–2 developers).**
  Shape: 2–3 people; trusted contributors; tight feedback loops; can tolerate manual hook-collision resolution and brownfield gap-report editing. Compressed Mode (3 HITL gates) is the implicit default. The Looplog example in IDEA-001 is the canonical scenario. Confirmed via GRILL-CRUX-002.
- **Secondary (value but not the trade-off pivot) — regulated-industry teams.**
  Healthtech, finance, etc. Audit-trail value is real to them; they will get it; v1.0 will not bend its UX or gate design to optimize for compliance-heavy workflows. The healthtech patient-data-export example in IDEA-001 illustrates the upside.
- **Secondary (value but not the trade-off pivot) — design-conscious teams.**
  DTC, brand-led. Design-reviewer agent and TOKENS / DESIGN_BRIEF artifacts work end-to-end; Gate 4.5 will not be optimized for design-led organizations at the expense of dev-led founding-team flow. The DTC PDP redesign example in IDEA-001 illustrates the upside.

- **REQ-CRUX-012** — `costs/log.csv` records every LLM agent invocation with `wall_seconds ≥ 60` (threshold pinned per SPEC-CRIT-002). (must)
- **REQ-CRUX-017** — test-writer / coder / reviewer are three separate subagent invocations with no shared transcript. (must)

- **REQ-CRUX-016** — Gate 5 is advisory; user accepts/edits/declines per item; `harness.lock` records `user_edited`. (must)
- **REQ-CRUX-022** — Hook collisions (same event + matcher, no priority) cause hard halt before `harness.lock`. (must)

### Module decomposition (Gate 3)

- **REQ-CRUX-018** — Amendments are layered AMD files; SKILL.md never mutated; orchestrator merges amendments into runtime briefs. (must)
