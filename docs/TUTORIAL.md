# CRUX Quickstart — End-to-end demo

This short tutorial reproduces the minimal local gate flow used in the demo:
init → grill → prd → modules → architect → harness → task → release-check.

Prerequisites
- A checked-out repository at C:\Dev\CRUX.
- Node (>=18), `pnpm` installed and dev dependencies installed (`pnpm install`).
- `npx` is available (bundled with Node).

Quick steps

1) Create a temporary workspace and enter it (PowerShell):

```powershell
$TUTORIAL_DIR = Join-Path $env:TEMP 'crux-tutorial-1'
New-Item -ItemType Directory -Path $TUTORIAL_DIR -Force | Out-Null
Set-Location $TUTORIAL_DIR
```

2) Initialize the minimal CRUX skeleton in the temp workspace:

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux init --greenfield
```

3) Add an IDEA file (example): create `docs/sdlc/input/IDEA-001.md` with YAML frontmatter. Example (PowerShell):

```powershell
$idea = @"
---
id: IDEA-001
ingested_at: 2026-05-06T12:00:00Z
source_path: brief.md
classification: brief
depth: surface
design_gate_enabled: false
---

Build a task tracker that uses AI to auto-prioritize work items based on velocity and deadlines.
"@
New-Item -ItemType Directory -Path docs\sdlc\input -Force | Out-Null
Set-Content -Path docs\sdlc\input\IDEA-001.md -Value $idea -Encoding utf8
```

4) Generate GRILL questions for the idea:

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux grill IDEA-001
```

5) Answer at least one GRILL (open docs/sdlc/grill/GRILL-*.yaml and fill `answer:` and set `state: answered`), or run PRD with `--force` to bypass thresholds. Then draft the PRD:

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux prd IDEA-001
```

6) Infer modules from the PRD:

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux modules PRD
```

7) Run architect scaffolding (creates ADRs / critiques / pre-mortem placeholders):

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux architect
```

8) Install harness non-interactively (generate `harness.lock`):

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux harness install --accept-defaults
```

9) Create a minimal task to exercise the task workflow (example file path):

docs/sdlc/tasks/TASK-001/TASK.yaml

Example content for `TASK.yaml`:

```yaml
id: TASK-001
title: Implement basic priority scoring
satisfies:
  - REQ-001
touches_files:
  - src/priority.ts
honors_adrs: []
```

10) Run the task lifecycle (test-writer → coder → reviewer) and produce `PR_DESCRIPTION.md`:

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux task TASK-001
```

11) Final release check:

```powershell
npx --prefix C:\Dev\CRUX tsx C:\Dev\CRUX\packages\cli\bin\crux release-check
```

Expected artifacts (workspace-relative)
- docs/sdlc/input/IDEA-001.md — original idea brief
- docs/sdlc/grill/GRILL-*.yaml — generated grill questions
- docs/sdlc/prd/PRD.md and docs/sdlc/prd/REQ-*.yaml — generated PRD and requirements
- docs/sdlc/modules/MOD-*.yaml — inferred modules
- docs/sdlc/adr/ADR-*.yaml — ADRs (scaffolded by architect)
- docs/sdlc/harness/harness.lock — harness install output
- docs/sdlc/tasks/TASK-001/PR_DESCRIPTION.md — PR description generated by task flow
